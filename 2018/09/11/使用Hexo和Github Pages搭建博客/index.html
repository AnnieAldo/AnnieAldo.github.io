使用Hexo和Github Pages搭建博客

按照常规，为了庆祝博客搭建成功，发一篇教程。
当然，也是因为我在搭建博客过程时废了一番功夫，
似乎目前没有什么教程能够让一个零基础的新手轻松搭建自己的博客。
因此写了这个教程，主要针对对hexo没有概念、第一次搭建博客的小白用户。

预备工作

预备知识

你需要熟悉Markdown语法，因为之后你的博客就需要用它来写

预备环境

请安装好git和node.js
如果不会，这些是能够轻松百度到的

第1步：安装Hexo

下述步骤和Hexo官方文档中一致。

走到这一步，复制命令窗口中的地址到浏览器中，你已经可以在本地运行自己的博客了。

第2步：Git

创建Repository

下述步骤和Github Pages官方文档中一致。

登录Github，创建自己的Repository

如图填写repository name为username.github.io，其中username必须是你Github的用户名

创建完成后，复制此时显示的url

配置本地博客

打开博客根目录下的_config.yml文件
修改Deployment下面的配置，把刚刚复制的url粘贴到repo:后面

这里给小白们解释一下，yml是一种配置语法
我们刚刚相当于把Github上你创建的repository配置到了这个hexo项目中
type: 我们选择Github Pages来生成博客
repo: 我们的Github repository地址
branch: 选择master也就是主线来配置博客

修改好了_config.yml文件之后，我们就可以将博客部署在Github Pages中了
在命令窗口进入博客根目录

这一步成功后，我们就可以在相应的Github Pages看到自己的博客了
博客的地址是https://username.github.io/
注意，如果你是第一次创建这个名称的Github Pages，进入页面会显示404
一般需要等待20-30min就可以正常显示了

以后你通过hexo g和hexo d更新博客时，一般也会有延迟，但就不会这么久啦
所以还是建议更新博客的时候，先在本地使用hexo s调试好之后，一次性提交到git，毕竟本地调试时没有延迟的

博客调试与更新

命令 作用
hexo s == hexo server 1. 从本地运行博客
hexo g == hexo generate 2. 生成待提交的博客静态文件
hexo d == hexo deploy 3. 将生成的博客提交到git

好了，在等待Github Pages刷新出来的这段时间里，让我们接着给我们的博客安装一个主题吧~

第3步：安装主题

官方博客可以说是很丑陋了，让我们一起来给自己的博客挑选一个主题吧~
在这里可以看到Hexo社区中的所有主题，点开Demo就能够预览

网上很多教程都会使用NexT主题，如果你也喜欢这款主题，可以搜索一下其他关于这个主题的教程

本篇教程中介绍我自己使用的fexo主题
这个主题非常简洁，拥有漂亮的Markdown语法渲染，而且自定义程度高
后文也会提到一些主题自定义的内容

下面的文章都会按照安装fexo主题的步骤去讲
你可以选择和我安装一样的主题，也可以选择只看主题间共性的内容

安装主题

在命令窗口进入博客根目录

启用主题

打开博客根目录下的_config.yml文件
修改Themes下面的配置

自定义主题

打开博客根目录下的themes/fexo/_config.yml，按照fexo文档中的讲解配置

这里给出我博客的完整配置，可以直接替换原版的themes/fexo/_config.yml文件

这个配置文件具有以下特性：
