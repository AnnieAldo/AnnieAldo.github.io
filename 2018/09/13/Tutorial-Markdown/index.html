环境配置

  1. 安装Atom（详见Tutorial-Atom）
  2. 使用插件：Markdown Preview Enhanced [官方文档]

Markdown Preview Enhanced设置

进入扩展的设置页：

  1. 设置样式为vue.css
  2. 勾选Break On Single New Line
  3. 设置Math Rendering Option为MathJax
  4. 勾选Enable extended table syntax使表格扩展语法能够使用
  5. 勾选Print Backgroud使样式在导出时生效
  6. 勾选Enable Script Execution使导出能够自动生成目录

其他环境配置

  • 修改系统样式
    • Win：直接everything搜索文件
    • Mac：文件路径为/Users/xxxxxxxx/.atom/packages/markdown-preview-enhanced/node_modules/@shd101wyy/mume/styles/preview_theme

## 目录生成
- 在Atom编辑器中调出命令窗口Command + Shift + P
- 输入Create Toc

如果你想要在TOC中排除一个标题，请在你的标题后面添加 {ignore=true} 即可。

相关参数：
- orderedList 是否使用有序列表。
- depthFrom, depthTo 规定所包含的标题等级，可设置为1~6。
- ignoreLink 如果设置为 true，那么 TOC 将不会被超链接。

## 导入文件


### 图片与表格
- .jpeg(.jpg) .gif .png .apng .svg .bmp 文件将会直接被当作 markdown 图片被引用。
- .csv 文件将会被转换成 markdown 表格。
### 画图
- .mermaid 将会被 mermaid 渲染。
- .dot 文件将会被 viz.js (graphviz) 渲染。
- .plantuml(.puml) 文件将会被 PlantUML 渲染。
### 内容
- .pdf 文件将会被 pdf2svg 转换为 svg 然后被引用。
- markdown 将会被分析处理然后被引用。
- 其他所有的文件都将被视为代码块。
### 页面
- .html 将会直接被引入。
- .js 将会被引用为 <script src="你的 js 文件"></script>
- .less 和 .css 将会被引用为 style。目前 less 只支持本地文件。 .css 文件将会被引用为 <link rel="stylesheet" href="你的 css 文件">。
